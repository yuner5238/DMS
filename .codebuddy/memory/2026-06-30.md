## 表格列宽优化
- 按建议调整 source（7%→6%）、responsible（4%→3%）、location（8%→7%），腾出 3% 给核心列
- 标签列默认 width: 10%, min-width: 160px；≤1500px 时 width: 180px（固定px）+ white-space: nowrap
- 根因：table-layout: fixed 中 th 的 min-width 不生效，需用固定 px width
- CSS 版本 v36，JS 版本 v33

## 标签列宽度与显示逻辑联动
- 用户要求：标签数量缩减时列宽也同步缩减，释放空间给设备名称/规格型号/来源/位置等核心列
- 最终方案：
  - \>1500px：标签 9%，来源 7%，标签显示 2 个 + "+n"
  - ≤1500px：标签 6%，来源 6%，标签显示 1 个 + "+n"
- 去掉所有 min-width（在 table-layout:fixed 中不生效且造成挤压失真）
- JS renderTagBadges(d, window.innerWidth < 1500 ? 1 : 2)
- CSS v47，JS v35

## 移动端视图修复
- **根本问题**：移动端 CSS 用 `attr(data-label)` 显示标签，但 `<td>` 从未设置 `data-label` 属性，导致卡片布局下每行无标签头
- **修复**：给所有 `<td>` 添加 `data-label`（设备ID/名称/序列号/规格型号/来源/数量/标签/部门/负责人/位置去向/状态/到期日/入库时间/备注/操作）
- **CSS 清理**：
  - 合并两个冲突的 `@media (max-width: 768px)` 块（侧边栏：移除旧版 stacking 布局规则，保留新版抽屉式 slide-in）
  - 移除遗留的 `.stats-row` / `.stat-card` / `.stat-card-value` 等不存在类的样式
  - 移动端表格卡片：`border-radius` 圆角 + `margin-bottom` 间隔 + `border` 轻边框
  - 无标签列 `data-label=""` 隐藏伪元素（批量勾选框）
  - 操作列去标签头、右对齐；按钮放大到 34px 触控区域
  - 隐藏列设置按钮（移动端不适用）
- CSS v48，JS v36

## 移动端侧边栏 z-index 修复
- **问题**：公告栏 `ann-bar-wrap`（z-index:101） > 侧边栏 `sidebar`（z-index:100），移动端滑出侧边栏时被公告栏遮挡
- **修复**：`@media (max-width:768px)` 中 sidebar 提升到 z-index:103，sidebar-overlay 提升到 z-index:102，均高于 ann-bar-wrap
- CSS v49
