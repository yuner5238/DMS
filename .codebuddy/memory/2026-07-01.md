# 2026-07-01

## 移动端 header-row 响应式优化（三轮）
- 根因：`@media (max-width: 1024px)` 下 `.header-row` 变为 `flex-direction: column`，而 `.search-container` 的 `flex: 1 1 180px` 在列方向下 flex-basis 作用于高度，导致搜索框至少 180px 高。

### 第一轮修复（搜索框高度 + toolbar 不换行）
- `.search-container { flex: 0 0 auto; }` — 高度由内容决定
- `.sidebar-toggle-btn { align-self: flex-start; }` — 防止 hamburger 被 stretch 到全宽
- `.toolbar { margin-left: 0; justify-content: flex-start; }` — 列布局下左对齐
- `@768px .toolbar { flex-wrap: nowrap; gap: 4px; }` — 按钮不换行
- `@768px .toolbar-btn { padding: 5px 10px; gap: 3px; font-size: var(--text-xs); }` — 收紧间距

### 第二轮优化（断点分层 + 平板横向保留）
- **列布局从 1024px 下沉到 768px**：平板(769-1024px) 保留横向 header-row，只收紧间距
- `@1024px`：content-header padding 32→20px，toolbar 按钮稍缩小(5px 12px)，gap 收至 5px
- `@768px`：列布局激活，header-row gap 12→8px，其余移动端卡片布局不变
- 桌面端(>1024px) 完全不受影响

### 第三轮优化（移动端搜索栏 + 筛选按钮）
- 在 `.search-container` 内新增 `#searchFilterBtn`（`bi-sliders` 图标），默认隐藏，仅在 `@768px` 显示
- 移动端隐藏 toolbar 中原 `#sortBtn`，由搜索栏右侧的筛选按钮统一触发排序下拉菜单
- `#sortDropdown` 从 `.toolbar` 内移到 `.header-row` 层级，JS 动态计算相对点击按钮的 `top/left`
- `#searchInput` 固定高度 36px，与筛选按钮等高，视觉统一
- 桌面端完全不受影响
