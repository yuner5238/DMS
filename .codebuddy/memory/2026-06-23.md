## 仓库列表追加在库设备数量
- **后端**：最终回归最简单的 `SELECT * FROM warehouses ORDER BY id`，不做任何 JOIN/COUNT
- **前端**：新增 `updateWarehouseCounts()` 在 `loadDevices()` 全部设备加载后统计各仓库在库数（`location_status !== 'checked_out'`），"全部仓库"汇总显示总和
- **修复间歇性 500**：TiDB 云端连接不稳定导致即使简单查询也偶尔失败 → 前端 `loadWarehouses()` 加 3 次重试（500ms 间隔）+ 后端 `query()`/`queryLarge()` 重试码扩展（新增 `PROTOCOL_ENQUEUE_AFTER_FATAL_ERROR`、`PROTOCOL_ENQUEUE_AFTER_QUIT`）+ 详细错误日志输出 code/message
- **修复 ECONNRESET 重试失效**：`pool.query()` 拿到的连接断开后坏连接放回池子，重试又拿到同一个。改为 `pool.getConnection()` → `conn.query()` 模式，出错时 `conn.destroy()` 销毁坏连接确保下次拿到新连接。重试间隔从 200ms 提升到 500ms（TiDB Serverless 冷启动恢复需更长时间）。`keepAliveInitialDelay` 设为 30000ms 保持连接活跃。`queryLarge()` 同步改造。
- **未分配仓库设备标红**：表格视图仓库列空时显示红色"未分配仓库"；卡片列表视图 meta 行显示红色"未分配仓库"标记
- **表格表头改为生产版 15 列**：设备ID、设备名称、序列号、规格型号、来源、数量、标签、所属路径、负责人、位置/去向、状态、到期日期、入库时间、备注、操作。新增 `escapeOnClick()`、`renderTagBadges()` 辅助函数，CSS 新增 `.device-id-badge`、`.device-id-badge-no-warehouse`、`.status-normal/abnormal/maintenance`、`.tag-more`、`.remark-icon` 等样式
- **表头/行与生产版完全对齐**：所有 `<th>` 和 `<td>` 添加 `data-col` 属性（device-id/name/serial-number/spec-model/source/quantity/tags/department/responsible/location/status/expiry/checkin/remark/actions），操作列 `<th class="actions-th"><span>操作</span></th>`。备注列改为 `remark-tooltip-wrapper` 点击弹出（新增 `showRemarkPreview` 函数）。按钮参数改用 `escapeOnClick()`。CSS 新增 `.remark-tooltip-wrapper`、`.actions-th` 样式
- **列设置功能**：操作列表头新增齿轮按钮 + 下拉菜单，实现列显示/隐藏控制。新增 `COLUMN_DEFS`、`columnVisibility`（localStorage 持久化）、`applyColumnVisibility()`、`toggleColumnSettings()`、`toggleColumn()` 函数。默认隐藏序列号/所属路径/负责人/状态/入库时间。CSS 新增 `.col-settings-btn`、`.column-settings-menu`、`.column-settings-item` 样式
- **列设置修复**：`.d-none` CSS 类缺失导致列隐显不生效 → 新增 `.d-none { display: none !important; }`。齿轮图标 `bi-gear` 改为 Unicode `⚙` 不依赖图标字体。CSS/JS 加 `?v=3` 版本号破除缓存。
